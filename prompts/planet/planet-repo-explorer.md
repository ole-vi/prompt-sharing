# Planet Repository Explorer

You are helping a new contributor understand the Planet Learning System codebase.

**Repository:** https://github.com/open-learning-exchange/planet

## Your Task

Analyze the Planet repository and provide detailed answers to help the new contributor understand the project structure, technology stack, and development practices.

## Questions to Answer

### 1. Project Overview
- What is the main purpose of the Planet application?
- What type of application is it (web app, mobile app, desktop)?
- What problem does it solve?

### 2. Technology Stack
- What is the primary programming language?
- What framework(s) are being used?
- What are the major dependencies? (Check package.json or similar)
- What database technology is used?
- Are there any notable libraries or tools?

### 3. Architecture & Structure
- What is the folder structure? Describe the purpose of major directories
- How is the code organized? (MVC, component-based, etc.)
- Where is the main entry point?
- How are routes/navigation handled?
- Where are API calls made?

### 4. Development Workflow
- What are the available npm/yarn scripts? (build, test, dev, etc.)
- How do you run the application locally?
- Are there any environment variables or configuration files needed?
- What testing framework is used?
- Is there a linting/formatting setup?

### 5. Key Features & Modules
- What are the main features of the application?
- What are the core modules or components?
- How is user authentication handled?
- What is the data synchronization mechanism?
- Are there any offline-first capabilities?

### 6. Contributing Guidelines
- Is there a CONTRIBUTING.md file? What does it say?
- What is the branch naming convention?
- What is the commit message format?
- Are there pull request templates or guidelines?
- What should a new contributor know before starting?

### 7. Common Gotchas & Important Notes
- Are there any known issues or limitations?
- What are common setup problems?
- Are there any deprecated patterns or code to avoid?
- What browser/platform compatibility considerations exist?

### 8. Resources for Learning
- Where is the documentation?
- Are there any design documents or architecture diagrams?
- Are there example implementations or sample code?
- Who are the key maintainers to reach out to?

## Output Format

Provide clear, detailed answers with:
- Code examples where relevant
- File paths and line numbers when referencing specific code
- Links to specific files in the repository
- Explanations suitable for someone new to the codebase

Focus on practical information that helps a new contributor start contributing quickly.
