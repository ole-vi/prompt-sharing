# myPlanet Repository Explorer

You are helping a new contributor understand the myPlanet mobile application codebase.

**Repository:** https://github.com/open-learning-exchange/myplanet

## Your Task

Analyze the myPlanet repository and provide detailed answers to help the new contributor understand the project structure, technology stack, and development practices.

## Questions to Answer

### 1. Project Overview
- What is the main purpose of the myPlanet application?
- What platforms does it target? (iOS, Android, both?)
- How does myPlanet relate to the Planet web application?
- What problem does it solve for users?

### 2. Technology Stack
- What is the primary programming language?
- What mobile framework is being used? (React Native, Flutter, Native, etc.)
- What are the major dependencies? (Check package.json, pubspec.yaml, or build.gradle)
- How does it handle local data storage?
- What networking/API libraries are used?

### 3. Architecture & Structure
- What is the folder structure? Describe the purpose of major directories
- How is the code organized? (feature-based, layer-based, etc.)
- Where is the main entry point for the application?
- How is navigation/routing handled?
- Where are API calls and data fetching implemented?
- How is state management handled?

### 4. Development Workflow
- What are the available build scripts? (build, test, dev, etc.)
- How do you run the application locally on iOS/Android?
- Are there any environment variables or configuration files needed?
- What testing framework is used?
- Is there a linting/formatting setup?
- How do you build for production/release?

### 5. Key Features & Modules
- What are the main features of the mobile app?
- What are the core screens or components?
- How is user authentication handled?
- What offline capabilities exist?
- How does data sync with Planet servers work?
- Are there push notifications or background tasks?

### 6. Mobile-Specific Considerations
- What permissions does the app require?
- How are native modules/plugins used?
- Are there platform-specific implementations (iOS vs Android)?
- How is app performance optimized?
- What accessibility features are implemented?

### 7. Contributing Guidelines
- Is there a CONTRIBUTING.md file? What does it say?
- What is the branch naming convention?
- What is the commit message format?
- Are there pull request templates or guidelines?
- What should a new contributor know before starting?
- How do you test on physical devices vs emulators?

### 8. Common Gotchas & Important Notes
- Are there any known issues or limitations?
- What are common setup problems? (SDK versions, dependencies, etc.)
- Are there any deprecated patterns or code to avoid?
- What device/OS compatibility considerations exist?
- Are there any build-specific issues (iOS certificates, Android signing)?

### 9. Resources for Learning
- Where is the documentation?
- Are there any design documents or architecture diagrams?
- Are there example implementations or sample code?
- Who are the key maintainers to reach out to?
- What related repositories should new contributors be aware of?

## Output Format

Provide clear, detailed answers with:
- Code examples where relevant
- File paths and line numbers when referencing specific code
- Links to specific files in the repository
- Platform-specific notes when applicable
- Explanations suitable for someone new to mobile development

Focus on practical information that helps a new contributor start contributing quickly and understand mobile-specific challenges.
